# Presentation Guide

Use this reference when the output is intended for a supervisor, advisor, senior-student discussion, research-group presentation, class presentation, or peer demo.

## First Sentence Disclaimer

Start the report or oral presentation with:

> This is a Draft Demo workflow scaffold. It shows the paper structure, method route, and replacement plan; it is not completed empirical findings.

## Recommended Presentation Order

1. **Boundary first** - State that simulated material is clearly labeled and not real data.
2. **Real input** - Show the real literature landscape, dataset, brief, or source material already available.
3. **Research route** - Explain why qualitative-led, questionnaire-led, mixed-methods, literature-led, or demo mode was chosen.
4. **Artifacts** - Show the research design, interview guide, optional questionnaire, codebook, PDF sample, and audit file.
5. **Risk control** - Explain where citations, ethics, participant data, or findings are still missing.
6. **Next work plan** - Present the minimum viable real-study plan with concrete targets.

## 30-Second Version

Use when the user needs a quick summary:

> I built a Draft Demo package to show what this social-science paper could become. The real part is the topic framing and available literature landscape; the simulated part is clearly marked. The package includes research questions, method design, interview and questionnaire tools, a demo paper PDF, and an audit checklist. The next step is not submission, but literature verification, ethics routing, pilot interviews, and replacing simulated findings with real evidence.

## Three-Minute Version

Use when the user needs a fuller advisor, senior-student, research-group, class, or peer-facing report:

1. "The goal is to turn a broad topic into a social-science research workflow."
2. "The current demo uses [real input] only for framing; all example participants/findings are labeled Draft Demo."
3. "The recommended route is [route] because the research question asks about [meaning/mechanism/experience/distribution]."
4. "The package contains [files/artifacts], so we can inspect the whole paper pipeline."
5. "The weak points are [literature verification / ethics / real interviews / coding / questionnaire validation]."
6. "The next realistic step is [minimum viable real-study plan]."

## Forbidden Claims

Do not say:

- "We already have findings."
- "The interviews show..."
- "The questionnaire is validated."
- "The literature proves this."
- "This is ready for submission."
- "AI completed the research."

Use instead:

- "The demo illustrates possible findings."
- "The interview protocol is ready for pilot testing."
- "The questionnaire is a construct-item draft."
- "The candidate literature needs verification."
- "This is a scaffold for advisor feedback."

## Presentation Report Template

Create a report with these sections:

1. One-sentence disclaimer.
2. What was produced.
3. Which parts are real, proposed, simulated, or TODO.
4. How each artifact serves the paper.
5. Main risks and how they are controlled.
6. Minimum viable real-study plan.
7. Recommended way to discuss the demo.
