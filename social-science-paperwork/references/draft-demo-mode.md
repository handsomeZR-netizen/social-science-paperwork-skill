# Draft Demo Mode

Use this reference when the user wants a classroom presentation, peer demo, boss report, sample manuscript, mock article, or complete draft that is explicitly not for real submission.

## Non-Negotiable Labeling

Every Draft Demo output must contain a visible notice near the top:

`Draft Demo Notice: This manuscript is for classroom/demo use only. Simulated participants, data, quotes, codes, and findings are illustrative and must not be presented as completed research or submitted as a real paper.`

Also label data-like sections:

- `Real input` - User-provided topic, literature notes, corpus findings, cases, or constraints.
- `Proposed design` - Methods that could be used in a real study.
- `Simulated material` - Mock participants, excerpts, survey patterns, codes, or findings.
- `Writing example` - Polished text demonstrating how the section could read.

## Output Package

For a complete demo, produce:

1. `demo_manuscript_draft.md` - Manuscript-like draft.
2. `demo_interview_protocol.md` - Semi-structured interview protocol.
3. `demo_questionnaire.csv` - Optional construct-item table.
4. `demo_codebook.csv` - Demo codebook with simulated examples labeled.
5. `demo_presentation_notes.md` - Notes explaining the workflow and limitations.

If the user only asks in chat, provide the same structure in the response or write files when requested.

## Manuscript Structure

Use this order:

1. Title.
2. Draft Demo Notice.
3. Abstract.
4. Keywords.
5. Introduction.
6. Literature Review and Conceptual Framework.
7. Research Questions.
8. Methodology as proposed design.
9. Illustrative Findings, clearly labeled simulated.
10. Discussion.
11. Conclusion.
12. Limitations and next real-data steps.
13. Appendices for protocol, questionnaire, and codebook.

## Simulated Findings Rules

Allowed in Draft Demo Mode:

- Mock participant IDs such as `P01-demo`.
- Short illustrative excerpts marked as simulated.
- Demonstration themes, codes, and matrices.
- Plausible descriptive survey patterns without fake precision.
- Example methods wording that says what a real study would do.

Not allowed:

- Claiming interviews, surveys, coding, ethics approval, or statistical tests were actually completed.
- Providing fabricated DOIs, citations, journal decisions, p values, alpha coefficients, or sample demographics as real.
- Removing the demo notice to make the draft look like a completed empirical paper.

## Findings Language

Prefer:

- "The following illustrative findings show how the analysis section could be organized after data collection."
- "In this demo, the simulated excerpt is used to demonstrate the evidence chain."
- "A real manuscript would replace these examples with anonymized transcript excerpts and documented coding decisions."

Avoid:

- "The interviews revealed..."
- "Participants reported..."
- "The survey results show..."
- "The study found..."

Unless real data has been provided and verified.

## Presentation Notes

For class or supervisor reports, include:

- What the skill helped decide.
- Which parts are real inputs.
- Which parts are simulated for demonstration.
- How the workflow would become a real study.
- Risks that would need to be solved before submission.
