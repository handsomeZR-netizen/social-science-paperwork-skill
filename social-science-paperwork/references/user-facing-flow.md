# User-Facing Flow

Use this reference when the user is new to social science or humanities writing, asks how to collaborate with AI, wants a complete workflow, or does not know what inputs are needed.

## Plain-Language Opening

Start by saying what will happen in user terms:

> I will first ask only the questions that change the research design. Then I will choose the method route, map evidence to claims, and produce the exact files you need. I will label anything that is proposed, simulated, or still missing.

Do not begin with a long methodology lecture.

## Six Questions To Ask First

Ask these in one short batch unless the answer is already obvious:

1. What is this for: real thesis/paper, proposal, class demo, boss report, or simulated showcase?
2. What is the topic and expected discipline or course/journal?
3. What evidence do you already have: papers, interviews, cases, survey data, images/artifacts, or only an idea?
4. Do you need Chinese, English, bilingual, Markdown, Word-style text, LaTeX, PDF, or PPT?
5. Are human participants involved, or will they be involved later?
6. What should the next output be: research design, literature review, interview guide, questionnaire, analysis, full draft, or audit?

If the user is impatient, answer with a route recommendation and state assumptions.

## Route Cards

After the first answers, present one recommended route and one backup route.

### Qualitative-Led Route

Use when the topic involves meaning, experience, interpretation, culture, education, design practice, organizations, identity, heritage, or cases.

Output contract:

- research questions
- concept/theory matrix
- sampling plan
- interview protocol
- case/material plan
- thematic analysis plan
- claim-evidence map

### Questionnaire-Led Route

Use only when constructs are clear enough to measure.

Output contract:

- construct map
- item table
- source/justification column
- response scale
- pilot-test plan
- reliability/validity plan
- analysis skeleton

### Mixed-Methods Route

Use when the user needs both mechanism explanation and attitude/distribution evidence.

Output contract:

- priority design: qualitative-first, quantitative-first, or convergent
- integration point
- instrument mapping
- risk list

### Literature/Theory-Led Route

Use when there is no field data yet or the paper is conceptual.

Output contract:

- search/screening plan
- literature matrix
- synthesis themes
- conceptual framework
- contribution statement

### Premium Draft Demo Route

Use when the user clearly says it is not for real submission and wants a complete, polished showcase.

Output contract:

- LaTeX/PDF manuscript package
- section files
- appendices
- visible Draft Demo disclosure
- simulated-material inventory
- quality audit
- demo-to-real checklist

## User-Friendly File Naming

Prefer clear names:

- `research_design.md`
- `literature_matrix.csv`
- `interview_protocol.md`
- `questionnaire_items.csv`
- `codebook.csv`
- `claim_evidence_map.md`
- `quality_audit.md`
- `demo_to_real_checklist.md`

For Premium Draft Demo, use:

`reports/social_science_premium_demo_paper/paper/`

## End Every Workflow With

1. What was produced.
2. Which parts are real, proposed, simulated, or TODO.
3. What the user should show to a teacher/advisor.
4. What must be replaced before real submission.
5. One short next-step recommendation.
